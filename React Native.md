## **Базові питання**

- Що таке React Native і чим він відрізняється від React?
це фреймворк для створення мобільних додатків під iOS та Android з використанням JavaScript і React.  Головна відмінність у тому, що **React** використовується для вебу і рендерить HTML у браузері, а **React Native** рендерить **нативні UI-компоненти** мобільних платформ, не використовуючи HTML і DOM.
- Як React Native рендерить UI?
React Native описує інтерфейс через JavaScript, після чого цей код перетворюється у **нативні компоненти платформи** (наприклад, `UIView` на iOS та `View` на Android).  
Рендеринг відбувається без HTML і браузера — напряму через нативний UI-шар.
- У чому різниця між React Native і Flutter?
React Native використовує **JavaScript/TypeScript** і рендерить **нативні компоненти ОС**, тоді як Flutter використовує **Dart** і малює інтерфейс власним графічним рушієм.  
У React Native вигляд залежить від платформи, а у Flutter UI виглядає однаково на всіх платформах.
- Чи є React Native “нативним” фреймворком?
React Native вважається **кросплатформним фреймворком з нативним рендерингом**.  
Він не є повністю нативним, як Swift чи Kotlin, але використовує нативні компоненти та дозволяє писати нативний код за потреби. **Нативний** — це те, що **створене спеціально для конкретної платформи** і працює **напряму з її системою**, без проміжних шарів.

## **Компоненти та UI**

- Які базові компоненти є в React Native? (`View`, `Text`, `Image`, `ScrollView`, `FlatList`)
У React Native замість HTML використовуються власні базові компоненти.  
`View` — універсальний контейнер для розмітки, аналог `div`.  
`Text` — компонент для відображення тексту.  
`Image` — використовується для показу зображень.  
`ScrollView` — контейнер із прокруткою для невеликої кількості контенту.  
`FlatList` — оптимізований компонент для відображення великих списків даних.

- У чому різниця між `ScrollView` і `FlatList`?
`ScrollView` рендерить **усі елементи одразу**, тому підходить лише для невеликих списків. 
`FlatList` рендерить **тільки видимі елементи**, що значно покращує продуктивність при роботі з великими списками.

- Як працює Flexbox у React Native?
Flexbox у React Native використовується для розташування елементів, але за замовчуванням напрямок — `flexDirection: column`.  
Доступні властивості `flex`, `justifyContent`, `alignItems`, `alignSelf`, які дозволяють керувати вирівнюванням і розмірами елементів.

- Чим стилі в React Native відрізняються від CSS?
У React Native стилі задаються через **JavaScript-обʼєкти**, а не CSS-файли.  
Підтримується лише частина CSS-властивостей, немає псевдокласів (`:hover`), `media queries` та каскадності стилів.

- Що таке `StyleSheet.create` і навіщо він потрібен?
`StyleSheet.create` використовується для створення стилів з перевіркою помилок і оптимізацією продуктивності.  
Він робить код більш читабельним і дозволяє повторно використовувати стилі без зайвих перерендерів. **Каскадність стилів** — це принцип у CSS, за яким **браузер визначає, який стиль застосувати до елемента**, якщо на нього діє кілька стилів одночасно. **якщо для одного елемента є кілька стилів — каскадність вирішує, який “переможе”**

- Як працює state в React Native?
State в React Native працює так само, як і в React для вебу. Це внутрішні дані компонента, які зберігають поточний стан UI.  
Коли state змінюється, компонент **перерендерюється**, і інтерфейс оновлюється відповідно до нового стану.

- Чим `useState` відрізняється від `useRef`?
`useState` використовується для даних, зміна яких повинна **викликати перерендер** компонента.  
`useRef` зберігає значення між рендерами, але **не викликає перерендер** при зміні, тому часто використовується для збереження посилань або технічних значень (таймери, попередні значення).

- Як працює lifecycle функціонального компонента?
  У функціональних компонентах lifecycle реалізований через хуки.  
Монтування відповідає `useEffect` з порожнім масивом залежностей, оновлення — `useEffect` зі залежностями, а демонтування — функція очищення, яка повертається з `useEffect`.
    
- Як обробляти side effects у React Native?
Side effects обробляються за допомогою `useEffect` або спеціальних бібліотек для роботи зі станом, таких як Redux middleware.  Усі побічні дії виконуються **поза рендером компонента**, щоб не порушувати чистоту UI-логіки.

- Що таке React Navigation?
React Navigation — це популярна бібліотека для керування навігацією між екранами в React Native.  Вона дозволяє створювати стекові, табові, drawer-навігації та обробляти переходи між екранами з передачею параметрів.

- Чим `stack`, `tab`, `drawer` навігація відрізняються?
Stack (стекова)** — переходи від екрана до екрана, як сторінки у браузері. Має кнопку «назад».
Tab (вкладки)** — екрани відображаються у вигляді вкладок внизу або зверху екрану. Перемикаєш вкладки без ререндеру стеку.
Drawer (шторка/меню)** — бокове меню, яке відкривається свайпом або кнопкою, щоб перейти до різних екранів.

- Як передавати параметри між екранами?
Параметри передаються через `navigation.navigate`:
`navigation.navigate('Details', { id: 42 });`
На екрані Details їх отримують через `route.params.id`.  

- Як повернутися назад програмно?
Для повернення назад використовується метод `navigation.goBack()`.  
Якщо потрібно повернутися на конкретний екран у стеку, можна використовувати `navigation.navigate('ScreenName')`.

## **Робота з даними**

- Як робити HTTP-запити в React Native?
Через `fetch` або сторонні бібліотеки, найчастіше `axios`, використовуючи async/await.
- `fetch` vs `axios`
`fetch` — вбудований, але потребує більше ручної обробки.  
`axios` — зручніший, має interceptors і автоматично парсить JSON.
- Як обробляти loading та error стани?
Через окремі state: `loading`, `error`, `data`, які змінюються під час запиту.
- Як кешувати дані?
Через React Query, Redux Toolkit або локальне збереження (AsyncStorage).
- Як працювати з async/await?
`async` робить функцію асинхронною, `await` чекає результат проміса без callback-ів.

## **AsyncStorage та зберігання**

- Що таке `AsyncStorage`?
Асинхронне key-value сховище для невеликих даних у React Native.
- Чим він відрізняється від `localStorage`?
`AsyncStorage` асинхронний і працює на мобільних, `localStorage` — синхронний і для web.
- Що не варто зберігати в `AsyncStorage`?
Великі обʼєкти, паролі, чутливі дані без шифрування.
- Які альтернативи `AsyncStorage` ти знаєш?
Великі обʼєкти, паролі, чутливі дані без шифрування.

## **Native можливості**

- Як працює доступ до камери, геолокації, файлів?
Через permissions і нативні API, які доступні через бібліотеки.
- Що таке Native Modules?
Це нативний код (Java/Kotlin, Swift/Obj-C), доступний з JavaScript.
- У яких випадках треба писати нативний код?
Коли немає готової бібліотеки або потрібна висока продуктивність.
- Що таке bridge у React Native?
Механізм звʼязку між JavaScript і нативним кодом.

## **Performance**

- Чому можуть бути лаги в React Native?
Через зайві ререндери, великі списки або важку логіку в JS-потоці.
- Як оптимізувати списки?
Використовувати `FlatList`, `keyExtractor`, `getItemLayout`.
- Для чого потрібні `memo`, `useCallback`, `useMemo`?
Для запобігання зайвих ререндерів.
- Як зменшити кількість ререндерів?
Мінімізувати state, використовувати мемоізацію.
- Що таке virtualized lists?
Списки, які рендерять тільки видимі елементи.
